# Text-to-LoRA: JTBD Node Classification Demo

Proof-of-concept applying [Text-to-LoRA (T2L)](https://github.com/SakanaAI/text-to-lora)
(Charakorn et al., ICML 2025, Sakana AI) to classify JTBD (Jobs-to-be-Done)
node types from consumer interview utterances.

## Experiment Design

| Stage | Method | What it tests |
|-------|--------|---------------|
| 0 | Zero-shot prompting | Mistral-7B-Instruct with structured prompt, no adapter |
| 1 | T2L zero-shot | Hypernetwork generates LoRA from task description alone |
| 2 | Fine-tuned LoRA (QLoRA) | Upper bound — what labelled data buys |

## Task

**Quote-level node type classification.** Given a verbatim span from an interview
utterance and its full-utterance context, classify it into one of 8 JTBD node types:

`pain_point` | `gain_point` | `emotional_job` | `solution_approach` |
`job_statement` | `job_context` | `job_trigger` | `social_job`

## Workflow

### 1. Generate the dataset (local, WSL)

```bash
cd /path/to/text-to-lora-demo
uv venv
uv pip install -r requirements.txt

# Reads from interview-system-v2/synthetic_interviews (default)
uv run python scripts/prepare_dataset.py

# Or with explicit path:
uv run python scripts/prepare_dataset.py --data-dir /path/to/synthetic_interviews
```

Outputs `data/processed/{train,val,test}.jsonl`.

### 2. Commit and push

```bash
git add data/processed/
git commit -m "initial dataset: 780 examples, 8 interviews"
git push
```

### 3. Open notebooks in Colab

Each notebook has an idempotent setup cell that:
- Clones / pulls this repo to `/content/text-to-lora-demo`
- Installs dependencies
- Adds `src/` to `sys.path`

**Secrets required** (add via Colab left sidebar → 🔑 icon):
- `HF_TOKEN` — Hugging Face token (for gated Mistral model)

**Notebook order:**
1. `00_data_prep.ipynb` — EDA, no GPU needed
2. `01_baselines.ipynb` — Stage 0 & 1, T4 GPU
3. `02_lora_finetune.ipynb` — Stage 2 fine-tuning, A100 recommended

## Data

Raw interview JSONs live in `interview-system-v2/synthetic_interviews` (not committed here).
Only JTBD interviews (`methodology: jobs_to_be_done*`) are included in the dataset.

Current dataset stats (8 interviews):
- ~780 quote-level examples
- 7–8 node types; `pain_point` (~37%) dominates; `social_job` (~1%) is sparse

## Running on Hugging Face Jobs

An alternative to Google Colab for Stage 2 training. No local GPU required.

### 1. Push dataset to Hub

```bash
uv run scripts/push_dataset_to_hub.py --repo michaelarutyunov/jtbd-classification
```

Creates a private HF Dataset with train/val/test splits. Requires `HF_TOKEN` in `.env`.

### 2. Submit the training job

```bash
hf jobs uv run \
    --flavor a10g-large \
    --timeout 2h \
    --secrets HF_TOKEN \
    scripts/train_qlora_hf_jobs.py
```

Set `DATASET_REPO` and `OUTPUT_REPO` in the script (or pass via `--env`) before submitting.

The script:
- Loads the Hub dataset
- Fine-tunes Mistral-7B-Instruct-v0.2 with QLoRA (4-bit, r=8, q_proj+v_proj)
- Streams metrics to [Trackio](https://huggingface.co/spaces/YOUR_HF_USERNAME/trackio)
- Pushes the trained adapter to Hub on completion

### 3. Monitor and check status

```bash
hf jobs ps                        # list jobs
hf jobs logs <job-id>             # stream logs
hf jobs inspect <job-id>          # job details
```

**Hardware requirements:** a10g-large (24 GB) or larger for Mistral-7B + 4-bit quant.

### 4. Evaluate the trained model

Once the training job completes, evaluate the model on the test set:

```bash
hf jobs uv run \
    --flavor a10g-small \
    --timeout 30m \
    --secrets HF_TOKEN \
    scripts/evaluate_qlora_hf_jobs.py
```

This will:
- Load the trained QLoRA adapter from the Hub
- Run inference on the test set (197 examples)
- Compute accuracy, macro F1, and confusion matrix
- Print comparison with Stage 0 (zero-shot) baseline

## Notes

- LoRA config (rank=8, alpha=16, targets=q_proj+v_proj) matches T2L's `adapter_config.json`
  exactly — Stage 2 vs Stage 1 comparison is apples-to-apples on adapter capacity
- T2L Stage 1 (`01_baselines.ipynb`) requires A100 (40 GB): `load_hypermod_checkpoint`
  internally loads a second copy of Mistral-7B; combined VRAM exceeds T4 (16 GB)
- `social_job` is present in some v2 interviews but absent from the original 7-type JTBD spec;
  kept in dataset to observe model behaviour on rare/novel classes
